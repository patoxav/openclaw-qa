Paso 1

Instalación de OpenClaw

# Instalar Node.js v24 con nvm
nvm install 24
nvm use 24

# Instalar OpenClaw globalmente
npm install -g openclaw

# Iniciar la interfaz
openclaw

Skill 1 de 3

🎭 Skill: Playwright QA

¿Cómo se crea?

# En OpenClaw TUI escribes:
"create a new skill called playwright-qa.
When I say 'corre los tests de QA',
run npx playwright test --headed
inside ~/openclaw-qa"

¿Cómo se usa?

"corre los tests de QA"
"run QA tests"
  • 🖥️ Browser Chromium se abre visible
  • 🧪 Tests E2E corren en pantalla
  • 📊 Reporte en terminal al finalizar
  • ⚡ Flag --headed para verlo en vivo
Skill 2 de 3

🐙 Skill: GitHub Issues Automáticos

Cuando un test falla, OpenClaw captura el error y crea un issue en GitHub automáticamente — sin intervención humana.
💥

Test falla

Playwright detecta el assertion error y captura el stack trace completo.

📋

Issue creado

GitHub CLI crea el issue con título, descripción y detalles del error en patoxav/openclaw-qa.

🔍

Trazabilidad total

Cada fallo queda documentado, asignado y con historial en GitHub.

Skill 3 de 3

💬 Skill: Notificación en Slack

# Script notify-slack.sh
WEBHOOK_URL="https://hooks.slack.com/..."
CHANNEL="#nuevo-canal"

payload=$(jq -n \
  --arg text "$MESSAGE" \
  '{text: $text}')

curl -X POST \
  -H 'Content-type: application/json' \
  --data "$payload" \
  "$WEBHOOK_URL"

✓ Mensaje enviado a #nuevo-canal

Setup en 5 pasos

  • 1️⃣ Crear workspace en Slack
  • 2️⃣ Crear app "OpenClaw QA" en api.slack.com
  • 3️⃣ Activar Incoming Webhooks
  • 4️⃣ Agregar webhook al canal
  • 5️⃣ Configurar skill en OpenClaw
